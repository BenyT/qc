简介

qc是一个mvvm双向绑定的框架，类似AngularJs。当大家跟着文档学习学习qc的时候，就会发现，qc功能非常的强大, 但比angular却简单许多。

mvvm框架性能一直是被人诟病的问题，在qc的前身grootjs的时候，我曾经在项目中遇到过这个问题。在我想了很多办法来优化它， 但是还是达不到我对性能的要求的时候，我沮丧了一段时间。一段时间过后我准备重写grootjs，也就是现在的qc。 现在的qc性能我认为已经比较理想了，对性能的优化主要做了一下工作：

  • 1.在首屏加载完之前不会操作dom，直接拼接为html字符串然后在一起用innerHTML加载到页面
  • 2.每个数据模型的属性都会和相应的dom建立映射关系，当数据模型改变时，直接找到相应的dom
  • 3.用自己写的表达式解析器代替“eval”，因为我发现这个函数很耗性能

这个框架写于2014年8；当时为什么写这个框架，原因想起来有点可笑：当时的我学了一段时间angular， 我发现angular对于我实在是太复杂了，我发现我根本学不会，我认为前端不应该这么复杂，所以就自己造了个轮子。

QC主要功能

  • 1.数据的双向绑定
  • 2.组件开发，支持组件嵌套
  • 3.自定义事件，这个我认为很重要。因为在移动端有很多手势，用框架接口可以很方便的定义成事件
  • 4.动画支持，尤其是移动端，生硬的数据变换太影响体验了，同样qc的动画接口也很简单
  • 5.自带ajax模块和commonjs模块
  • 6.框架集成了常用的动画很移动端手势，当然大家可以用自己定义的：）
